# MakerWorld 本地归档小应用 v5.0 更新说明

## 1. 归档生成机制统一（模板驱动）
v5.0 将归档页生成逻辑统一为“基于在线模板输出离线页”。
- 本地归档页面不再依赖多套分散模板，改为复用在线 `model.html` 及其样式脚本体系。
- 后续若调整在线模型页样式，可通过重建机制快速同步到已归档页面。

## 2. 同模型二次归档改为“更新”语义
针对同一模型地址重复归档的场景，流程由“可能重复创建”改为“覆盖更新”：
- 自动识别已存在模型目录并更新内容（而非新增重复目录）。
- 同步更新采集时间与相关元数据。
- API 与前端提示统一为“模型已更新成功”。

## 3. 元数据增强：新增 `update_time`
`meta.json` 增加明确的 `update_time` 字段，用于标记最近一次更新时刻，便于排序、审计和后续自动化处理。

## 4. 已归档页面重建能力
新增“历史归档页快速同步”能力：
- 接口：`POST /api/archive/rebuild-pages`
- 配置页新增“更新已归档页面”按钮，一键按最新模板重建历史归档页面。
- 兼容旧版数据重建场景，可在保留 `index_v1.0.html` 的同时生成新版 `index.html`。

## 5. 插件体系整理（plugin 目录）
插件统一收敛到 `plugin/` 目录，并完善使用文档：
- `plugin/chrome_extension/mw_quick_archive_ext`：Chrome 插件
- `plugin/tampermonkey/mw_quick_archive.user.js`：油猴脚本
- 新增两份独立使用说明文档，方便部署与分发。

### Chrome 插件增强
- API 地址保存与连接测试
- 一键归档模型
- 自动 Cookie 同步 + 手动 Cookie 设置（选项页）
- 本地模型主页/归档结果快速打开

### 油猴脚本调整
- 去除自动 Cookie 抓取，保留手动设置与同步
- 归档按钮样式统一优化（避免图标与文本重叠）

## 6. 维护脚本 update.sh 优化
- 增加脚本头部详细说明（用途、流程、前置条件、退出码）。
- `git pull` 无更新时，提示是否继续重新部署（默认否）。
- `git pull` 执行失败时直接报错退出，避免静默失败。
