# v5.1 更新日志

> 更新日期: 2026-03-03

## 更新概述
本次更新聚焦手动导入与打印配置导入体验，完善 3MF 识别回显能力、导入命名规范、来源分类与持久化配置策略。

## 核心更新

### 1. 手动导入增强（识别与回显）
- 手动导入弹窗在“识别并填充”后，新增回显内容：
  - 封面与模型主信息
  - 设计图片缩略图
  - 附件文件列表
  - 实例配置列表（支持逐项启用/编辑）
- 新增“识别实例信息”能力：
  - 对“添加实例文件”中的 3MF 批量解析
  - 仅识别实例维度信息（标题、介绍、配置图片、盘图）
  - 不覆盖模型主信息

涉及文件:
- `app/templates/gallery.html`
- `app/static/js/manual_import.js`
- `app/static/css/manual_import.css`
- `app/three_mf_parser.py`

### 2. 模型详情页“添加打印配置”改为弹窗流程
- 原“直接识别并追加”按钮改为弹窗交互：
  1. 上传 3MF
  2. 识别并回显（标题、介绍、配置图、分盘图）
  3. 用户修改后保存
- 保存时调用原追加接口，兼容现有数据结构与页面渲染逻辑。

涉及文件:
- `app/templates/model.html`
- `app/static/js/model.js`
- `app/static/css/model.css`

### 3. 3MF 导入命名与来源规范
- 去除实例导入流程中的 `s01_` 前缀展示与落盘干扰：
  - 页面显示优先使用 `sourceFileName`
  - 归档保存统一使用清洗后的真实文件名
- 手动导入模型 `meta.source` 默认从 `others` 调整为 `LocalModel`。
- 画廊来源聚合规则调整：
  - `LocalModel_*` 与历史 `Others_*` 统一归类为“手动导入”
  - 来源筛选显示文案更新为“手动导入”

涉及文件:
- `app/server.py`
- `app/static/js/gallery.js`
- `app/static/js/model.js`

### 4. 手动导入编号持久化优化
- `manual_local_model_counter` 从 `app/config.json` 移除。
- 新增数据目录独立计数文件用于持久化：
  - `app/data/_manual_import_counter.json`
- 避免因配置覆盖导致编号回退；编号保持 6 位自增（`000001`）。

涉及文件:
- `app/server.py`
- `app/config.json`

## 兼容性说明
- 旧的 `Others_*` 数据目录继续可用。
- 历史手动导入模型在来源筛选中会归到“手动导入”。
- 离线打开（`file://`）场景仍遵循仅浏览/读取，上传类能力需通过本地服务访问。

## 验证建议
1. 在配置页执行一次手动导入，确认生成目录为 `LocalModel_00000X_*`。
2. 在手动导入弹窗检查设计图/附件/实例识别回显是否完整。
3. 在模型详情页测试“添加打印配置”弹窗：识别、编辑、保存是否生效。
4. 在首页来源筛选中确认“手动导入”分类可正常过滤新旧手动导入模型。
