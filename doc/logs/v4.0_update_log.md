# MakerWorld 本地归档小应用 v4.0 更新说明

## 核心架构大重构：转向客户端动态渲染
以往的模型详情页 (`model.html`) 完全依赖 Python 后端通过 Jinja2 模板进行服务端渲染 (SSR)。一旦遇到页面内如实例信息、图片列表等复杂的动态变更需求，维护起来代价极高。
在 v4.0 版本中，我们彻底移除了后端对 `model.html` 的数据注入，改为**纯静态输出页面骨架**。 
- **全新的 API**：增加了 `/api/models/{model_dir}/data` 数据接口，前端在页面加载后主动获取该模型的全部元数据（`meta.json`）。
- **客户端渲染引擎**：在 `model.js` 中重写了整套渲染逻辑 (`buildInstanceHtml`, 等)，依靠浏览器进行 DOM 的动态重写和拼装。这样极大地提升了前端的交互灵活性，并为今后增加更多像“分盘弹窗”这样复杂的层叠交互打下基础。

## Bambu Studio 拉起集成与 URL 转码机制
修复了长久以来调用本地 Bambu Studio 无法正常打开包含特殊字符（例如中文字符和空格）的 3mf 文件的问题。
- Bambu Studio 对传入 URL 的自解析逻辑极不支持标准的 URL Encode，常报出 `Invalid HTTP request received`。
- **Hex 转码策略**：我们在前端 `model.js` 构建调用链接时，将相对文件路径转换为 Hex 16 进制字符串。
- **自定义路由代理**：在 `server.py` 中新增 `GET /api/bambu/download/{hex_path}.3mf` 路由拦截请求。这样发给切片软件的链接既避开了繁杂的字符集解析，同时其伪装的 `.3mf` URL 尾缀也直接满足了 Bambu Studio 的本地格式校验逻辑，彻底解决了“文件格式无法识别”和“导入失败”等报错。

## 实例详情与分盘弹窗 UI 深度优化
响应用户需求，极大改进了模型详情页中“打印实例”(Instances) 的信息展示，高度还原官方原版视觉规范。
- **操作按钮重排**：隐藏了过于抢眼的下载/打印独立横杠，将它们与新增的“详情”按钮缩编为一行。通过 `flex-wrap: nowrap` 和内联滚动，即使在窄屏幕的手机设备上，这几个核心交互按钮也能整齐地靠右对齐。
- **剥离分盘信息**：原先直接塞在卡片底部的重量/时长/分盘预览图被彻底剥离，减轻了卡片阅读负担。
- **分盘详情弹窗 (Modal)**：按下“详情”按钮后，屏幕居中弹出半透明遮罩 Modal 详解。
- **双模响应式图文交互**：在 PC 宽屏下，弹窗分为左右两块：右侧是可滚动的盘片列表记录（盘注、时长、耗材重量），左侧是独立的图片交互预览窗（点击任意盘片列表项即可快速切换显示其大图，再次点击还可触发无损灯箱放大）。
  - 在列表项里，我们还纯手工复刻了官方的**耗材线轴 (Spool) 矢量图标**，让界面观感更加细腻紧凑。

## 其他更新
- 增加了 `model.html` / `gallery.html` 等静态资源的 Cache Busting (增加 `?v=2.6` 参数)，彻底杜绝了发版后用户浏览器还在使用旧样式文件的痛点问题。
- 灯箱层级 (`z-index`) 提升至 10000 级，确保其叠加在所有的原生窗口之上。
